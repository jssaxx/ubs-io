# 推理三级池化场景安装部署指南

本文档说明推理三级池化场景下 UBSIO-BoostIO 的构建、部署和使用方法。该场景只需要交付运行库，构建完成后使用 `ubsio-boostio/dist/lib` 目录即可。

## 适用场景

推理三级池化场景通常由推理服务进程加载 UBSIO 运行库，通过配置文件初始化本地缓存能力。部署时不需要完整 `boostio` 运行目录，也不需要单独安装命令行工具；只需要将 `dist/lib` 下的动态库放到推理服务可访问的位置。

## 构建环境准备

构建节点需要具备以下基础工具。

| 工具 | 说明 |
| --- | --- |
| `bash` | 执行构建脚本。 |
| `cmake` | 当前工程要求最低版本为 `3.16`。 |
| `make` | 编译 C/C++ 目标。 |
| `gcc/g++` | 编译 C/C++ 代码。 |
| `git` | 在线构建或准备离线三方源码时使用。 |

源码目录需要保持以下相对位置：

```text
ubs-io/
├── ubsio-boostio/
├── ubsio-kv/
└── ubsio-common/
```

## 在线构建

在可以访问三方源码仓的构建环境中执行：

```bash
cd ubsio-boostio
bash build.sh -t release --pkg
```

构建完成后，运行库会输出到：

```text
ubsio-boostio/dist/lib
```

该目录即为推理三级池化场景需要交付的库包目录。

## 离线构建

客户环境无法访问公网时，需要先在有网环境准备三方源码，再将源码和工程一起拷贝到客户环境编译。

### 有网环境准备三方源码

```bash
cd ubsio-boostio
bash build.sh --prepare_3rdparty
```

准备完成后，将 `ubsio-boostio/3rdparty` 中的三方源码随工程代码一起带到客户环境。该步骤也会准备 ubs-comm 构建所需的内部依赖源码。也可以打包三方源码：

```bash
cd ..
mkdir -p ubsio-boostio/dist
tar --exclude-vcs \
    --exclude='*/build' --exclude='*/build/*' \
    --exclude='*/dist' --exclude='*/dist/*' \
    --exclude='*/obj' --exclude='*/obj/*' \
    --exclude='*.o' --exclude='*.a' --exclude='*.so' --exclude='*.so.*' \
    -czf ubsio-boostio/dist/BoostIO_3rdparty_sources.tar.gz \
    -C ubsio-boostio \
    3rdparty/libboundscheck/libboundscheck \
    3rdparty/libaio/libaio \
    3rdparty/spdlog/spdlog \
    3rdparty/ubs-comm/ubs-comm
```

### 客户环境编译库包

在客户环境解压三方源码后执行：

```bash
cd ubsio-boostio
bash build.sh -t release --pkg
```

构建脚本会自动检测工程目录中的三方源码；源码齐全时，会直接使用本地源码构建。

最终仍只需要交付：

```text
ubsio-boostio/dist/lib
```

## 库包内容

`dist/lib` 中包含推理进程运行所需的 UBSIO 相关库，常见文件如下。

| 文件 | 说明 |
| --- | --- |
| `libbio_sdk.so` | SDK 动态库，推理服务侧主要加载入口。 |
| `libbio_server.so` | 本地缓存 server 运行库，单机 direct-call 场景会被 SDK 加载。 |
| `libbio_common.so` | 公共基础库。 |
| `libbio_security.so` | 安全相关公共能力。 |
| `libbio_underfs.so` | UnderFS 后端访问能力。 |
| `libhtracer.so` | trace 相关运行库。 |
| `libboundscheck.so` | 安全函数三方依赖。 |
| `libaio.so*` | 异步 I/O 三方依赖。 |
| `libubsio_kvc.so*` | KVCache C 接口库，默认构建 KV 组件时生成。 |

实际文件列表以当前构建产物为准。

## 部署库包

将 `dist/lib` 拷贝到推理服务节点，例如：

```bash
mkdir -p /opt/ubsio/lib
cp -a ubsio-boostio/dist/lib/* /opt/ubsio/lib/
```

推理服务启动前，需要先设置动态库搜索路径：

```bash
export LD_LIBRARY_PATH=/opt/ubsio/lib:${LD_LIBRARY_PATH}
```

如果业务使用其他安装目录，将 `/opt/ubsio/lib` 替换为实际的 `dist/lib` 拷贝路径。

## 配置文件

BoostIO 支持默认配置。未配置 `UBSIO_BIO_CONFIG_PATH` 时，推理进程会使用内置默认配置完成初始化。

如需覆盖默认配置，可以准备自定义配置文件，例如：

```text
/etc/ubsio/bio.conf
```

单机推理三级池化场景可参考 [UBSIO 单机模式配置参考](../单机模式配置说明.md)。常用配置项如下。

| 配置项 | 说明 |
| --- | --- |
| `bio.disk.path` | 缓存盘或分区路径。无盘场景可留空，仅使用内存缓存。 |
| `bio.mem.size_in_gb` | 单个进程使用的内存缓存容量。 |
| `bio.standalone.device_count` | 当前推理服务中的本地 server 进程数量。 |
| `bio.cache.mem_read_write_ratio` | 推理三级池化写缓存场景通常配置为 `0:10`。 |
| `bio.cache.disk_read_write_ratio` | 有盘写缓存场景通常配置为 `0:10`。 |
| `bio.underfs.file_system_type` | 本地单机场景使用 `none`。 |
| `bio.net.tls.enable.switch` | 本地单机无 TLS 文件时配置为 `false`。 |

需要使用自定义配置文件时，通过环境变量指定配置文件路径：

```bash
export UBSIO_BIO_CONFIG_PATH=/etc/ubsio/bio.conf
```

## 推理服务使用方式

推理服务启动前需要设置动态库搜索路径：

```bash
export LD_LIBRARY_PATH=/opt/ubsio/lib:${LD_LIBRARY_PATH}
```

如果需要使用自定义配置文件，再设置 `UBSIO_BIO_CONFIG_PATH`：

```bash
export UBSIO_BIO_CONFIG_PATH=/etc/ubsio/bio.conf
```

然后按业务原有方式启动推理服务。推理进程加载 UBSIO 相关库后，如果设置了 `UBSIO_BIO_CONFIG_PATH`，会读取该环境变量指向的配置文件；如果未设置，则使用默认配置。

示例：

```bash
export LD_LIBRARY_PATH=/opt/ubsio/lib:${LD_LIBRARY_PATH}
# 可选：使用自定义配置时设置
# export UBSIO_BIO_CONFIG_PATH=/etc/ubsio/bio.conf

# 按业务实际命令启动推理服务
bash start_inference.sh
```

## 常见问题

### 启动时报动态库找不到

检查推理服务启动脚本中是否设置了 `LD_LIBRARY_PATH`，并确认路径指向实际部署的 `dist/lib` 目录。

```bash
echo "${LD_LIBRARY_PATH}"
ls /opt/ubsio/lib
```

### 指定自定义配置后提示配置文件不存在

不配置 `UBSIO_BIO_CONFIG_PATH` 时会使用默认配置。只有设置了该环境变量时，才需要确认该路径对推理服务运行用户可读。

```bash
echo "${UBSIO_BIO_CONFIG_PATH}"
ls -l "${UBSIO_BIO_CONFIG_PATH}"
```

### 客户环境无法联网

不要在客户环境执行 `bash build.sh --prepare_3rdparty`。该命令需要访问三方源码仓。请在有网环境提前准备三方源码，再在客户环境使用：

```bash
bash build.sh -t release --pkg
```

生成的 `dist/lib` 目录就是最终交付给推理三级池化场景的库包。
